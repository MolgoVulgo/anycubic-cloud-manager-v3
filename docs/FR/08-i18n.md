# Internationalisation

Statut : `PARTIEL`.

## Objectif

Supporter anglais et français sans mélange entre texte UI, messages utilisateur, logs techniques et diagnostics. L’anglais est la langue de documentation par défaut ; le français reste maintenu.

## Catégories

| Type | Sens | Traitement |
| --- | --- | --- |
| A | UI statique | Catalogues/traduction obligatoire. |
| B | Message dynamique utilisateur | Template traduisible + paramètres. |
| C | Labels modèles visibles | Localisation ou mapping display. |
| D | Technique/debug | Peut rester technique. |
| E | Ambigu | Classer avant modification. |

## Règles architecture

Chaînes visibles QML traduisibles. Backend sans concaténation visible finale. Valeurs dynamiques en placeholders. Logs séparés des messages UI. Codes/protocol keys restent stables.

## Terminologie

Cloud file/Fichier cloud, Printer/Imprimante, Signed URL/URL signée, Vat/Cuve, Bottle/Bouteille, Preheating/Préchauffe, Print job/Tâche d’impression.

## Lots migration

UI statique, statuts QML, labels modèles, messages backend visibles, terminologie/catalogues, tests/régressions.

## Validation

Aucun nouveau texte visible hors traduction, accents UTF-8 corrects, placeholders pour valeurs dynamiques, logs séparés, équivalence fonctionnelle EN/FR.

## Décision

L’i18n est un contrat UI, pas une finition. Toute nouvelle UI production doit être classée et traduite dès son introduction.
